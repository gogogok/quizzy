# Quizzy — интерактивная платформа для проведения квизов

## Краткая пояснительная записка

Quizzy — веб-приложение для создания и проведения интерактивных квизов в реальном времени. Организатор может подготовить квиз, добавить вопросы и варианты ответов, запустить игровую сессию и показать итоговый рейтинг. Участники подключаются к сессии по коду комнаты, отвечают на вопросы и получают баллы с учётом правильности и скорости ответа.

## Цель проекта

Цель проекта — создать понятный и доступный сервис, который позволяет быстро организовать учебный или развлекательный квиз без установки дополнительного программного обеспечения. Продукт рассчитан на преподавателей, ведущих мероприятий, организаторов командных активностей и участников.

Основные задачи:

- регистрация и авторизация пользователей;
- создание, редактирование и удаление квизов;
- поддержка вопросов с одним или несколькими правильными ответами;
- добавление изображений к вопросам;
- подключение участников по коду комнаты;
- синхронное проведение игры в реальном времени;
- автоматический подсчёт баллов;
- отображение рейтинга и истории участия.

## Выбор средств разработки

### Клиентская часть

Клиент разработан на **React**. Компонентный подход упрощает повторное использование элементов интерфейса и дальнейшее развитие проекта. Для сборки используется **Vite**, поскольку он обеспечивает быстрый запуск среды разработки и компактную production-сборку. Навигация между страницами реализована с помощью **React Router**.

Для оформления используется собственный CSS без тяжёлой библиотеки компонентов. Это позволило точнее воспроизвести макеты, сохранить единый визуальный стиль и уменьшить количество внешних зависимостей.

### Серверная часть

Сервер создан на **Node.js** и **Express**. Этот стек использует JavaScript на клиенте и сервере, упрощает совместную разработку обеих частей и подходит для REST API.

Обновление состояния игры в реальном времени реализовано через **Socket.IO**. В отличие от периодических HTTP-запросов, постоянное соединение позволяет без заметной задержки сообщать участникам о начале вопроса, завершении времени и переходе к результатам.

Для проверки входных данных используется **Zod**. Пароли хешируются с помощью **bcryptjs**, а авторизация выполняется через JWT-токены.

### Хранение данных

В качестве СУБД выбрана **PostgreSQL**: она надёжно хранит связанные сущности пользователей, квизов, вопросов, игровых сессий, ответов и результатов. Работа с базой организована через **Prisma ORM**, которая предоставляет типизированную модель данных и управляемые миграции схемы.

### Развёртывание

Рабочая версия размещена на **Render**. Конфиденциальные параметры подключения к базе данных и ключи авторизации передаются через переменные окружения и не хранятся в репозитории.

## Основные этапы разработки

1. **Исследование и проектирование.** Были определены две основные роли — организатор и участник, описаны пользовательские сценарии и структура будущего приложения.
2. **Создание макетов.** В Figma подготовлены основные экраны, навигация, формы авторизации, редактор квиза, игровая комната и таблица результатов.
3. **Проектирование базы данных.** Созданы модели пользователей, квизов, вопросов, вариантов ответов, сессий, участников и результатов.
4. **Разработка серверного API.** Реализованы регистрация, авторизация, операции с квизами, создание игровых сессий и получение истории игр.
5. **Разработка интерфейса.** Макеты перенесены в React-компоненты, добавлены формы, валидация, адаптивные состояния и обработка ошибок.
6. **Реализация игры в реальном времени.** Настроено подключение по коду комнаты, синхронное переключение вопросов, таймер, отправка ответов и обновление рейтинга.
7. **Тестирование и исправление ошибок.** Проверены основные пользовательские маршруты, корректность начисления баллов, повторное подключение участников и работа приложения при нескольких одновременных соединениях.
8. **Развёртывание.** Клиент, сервер и PostgreSQL подготовлены к production-окружению и опубликованы на Render.

## Локальный запуск

Сначала клонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/gogogok/quizzy.git
cd quizzy
```

### 1. База данных

Команда выполняется из корневой папки проекта:

```bash
docker compose up -d
```

### 2. Сервер

```bash
cd server
cp .env.example .env
npm install
npm run db:generate
npm run db:deploy
npm run dev
```

Для локальной базы пароль в `DATABASE_URL` должен соответствовать значению из `docker-compose.yml`:

```env
DATABASE_URL=postgresql://quizzy:quizzy_local_password@127.0.0.1:5433/quizzy?schema=public
```

### 3. Клиент

В отдельном терминале перейдите в папку клонированного проекта, а затем в `client`:

```bash
cd quizzy/client
cp .env.example .env
npm install
npm run dev
```

После запуска приложение доступно по адресу [http://localhost:5173](http://localhost:5173).
